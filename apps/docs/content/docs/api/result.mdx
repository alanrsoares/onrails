# @onrails/result API Reference

## Core

### bimap <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Transform both tracks at once — `Ok` via `onOk`, `Err` via `onErr`.
Equivalent to `mapErr(onErr)(map(onOk)(result))` but in one pass.

```typescript
function bimap<T, U, E, F>(result: Result<T, E>, onOk: (value: T) => U, onErr: (error: E) => F): Result<U, F>
function bimap<T, U, E, F>(onOk: (value: T) => U, onErr: (error: E) => F): (result: Result<T, E>) => Result<U, F>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |
| `onOk` | `(value: T) => U` |  |
| `onErr` | `(error: E) => F` |  |

#### Example

```ts
bimap(parsed, (cfg) => cfg.name, (e) => ({ kind: "input", cause: e }));
```

### err <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Lifts a value into the error track.

```typescript
function err<T = never, E = unknown>(error: E): Result<T, E>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `error` | `E` |  |

#### Example

```ts
const r = err({ kind: "parse", message: "bad json" });
// Result<never, { kind: "parse"; message: string }>
```

### flatMap <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Canonical bind (Fantasy Land `chain`). Chains a Result-returning step,
widening the error union to `E | F`. Short-circuits on `Err`.

```typescript
function flatMap<T, U, E, F>(result: Result<T, E>, fn: (value: T) => Result<U, F>): Result<U, E | F>
function flatMap<T, U, F>(fn: (value: T) => Result<U, F>): <E>(result: Result<T, E>) => Result<U, F | E>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |
| `fn` | `(value: T) => Result<U, F>` |  |

#### Example

```ts
flatMap(parseInput(raw), (data) =>
  data.id != null ? ok(data) : err({ kind: "missing_id" as const }),
);
// Result<Data, ParseError | { kind: "missing_id" }>
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### flow <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Variadic point-free composition — chains up to nine unary fns left-to-right
into one reusable function. The first fn may take multiple arguments; the
rest are unary. Use [pipe](#pipe-ƒ) from `@onrails/result` when you already
have a starting value; use [flow](#flow-ƒ) to define a reusable composed
function with no value yet.

```typescript
function flow<A extends readonly unknown[], B>(ab: (...a: A) => B): (...a: A) => B
function flow<A extends readonly unknown[], B, C>(ab: (...a: A) => B, bc: (b: B) => C): (...a: A) => C
function flow<A extends readonly unknown[], B, C, D>(ab: (...a: A) => B, bc: (b: B) => C, cd: (c: C) => D): (...a: A) => D
function flow<A extends readonly unknown[], B, C, D, E>(ab: (...a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E): (...a: A) => E
function flow<A extends readonly unknown[], B, C, D, E, F>(ab: (...a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E, ef: (e: E) => F): (...a: A) => F
function flow<A extends readonly unknown[], B, C, D, E, F, G>(ab: (...a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E, ef: (e: E) => F, fg: (f: F) => G): (...a: A) => G
function flow<A extends readonly unknown[], B, C, D, E, F, G, H>(ab: (...a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E, ef: (e: E) => F, fg: (f: F) => G, gh: (g: G) => H): (...a: A) => H
function flow<A extends readonly unknown[], B, C, D, E, F, G, H, I>(ab: (...a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E, ef: (e: E) => F, fg: (f: F) => G, gh: (g: G) => H, hi: (h: H) => I): (...a: A) => I
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `ab` | `(...a: A) => B` |  |

#### Example

```ts
const parseUserName = flow(
  (raw: string) => parseConfig(raw),
  map((cfg) => cfg.user),
  flatMap((u) => (u.name ? ok(u.name) : err({ kind: "missing" as const }))),
);

parseUserName(raw); // Result<string, ParseError | { kind: "missing" }>
```

### map <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Transform the `Ok` value, passing `Err` through unchanged. Dual-form:
call data-first or curried (for use with [pipe](#pipe-ƒ)).

```typescript
function map<T, U, E>(result: Result<T, E>, fn: (value: T) => U): Result<U, E>
function map<T, U>(fn: (value: T) => U): <E>(result: Result<T, E>) => Result<U, E>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |
| `fn` | `(value: T) => U` |  |

#### Example

```ts
map(ok(2), (n) => n * 3);          // Ok 6 — data-first
pipe(ok("x"), map((s) => s.length));// Ok 1 — curried
```

### mapErr <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Transform the `Err` value, passing `Ok` through unchanged. Useful for
unifying heterogeneous failure types into one app-level union.

```typescript
function mapErr<T, E, F>(result: Result<T, E>, fn: (error: E) => F): Result<T, F>
function mapErr<E, F>(fn: (error: E) => F): <T>(result: Result<T, E>) => Result<T, F>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |
| `fn` | `(error: E) => F` |  |

#### Example

```ts
type AppError = { kind: "http"; status: number } | { kind: "parse" };
pipe(
  fetchSync(url),                                     // Result<Body, { status: number }>
  mapErr((e): AppError => ({ kind: "http", status: e.status })),
);
```

### match <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Terminal collapse — fold both tracks into a single value. Dual-form:
3-args data-first, 2-args curried for [pipe](#pipe-ƒ). Returns whatever
the handlers return.

For files that also import `match` from `ts-pattern`, use a namespace
import (`import * as R from "@onrails/result"` → `R.match`) to dissolve
the collision.

```typescript
function match<T, E, U>(result: Result<T, E>, onOk: (value: T) => U, onErr: (error: E) => U): U
function match<T, E, U>(onOk: (value: T) => U, onErr: (error: E) => U): (result: Result<T, E>) => U
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |
| `onOk` | `(value: T) => U` |  |
| `onErr` | `(error: E) => U` |  |

#### Example

```ts
const html = match(parsed, (cfg) => render(cfg), (e) => renderError(e));
```

### pipe <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Variadic value-first pipe — threads `value` through up to nine unary fns,
left-to-right. Use [pipe](#pipe-ƒ) when you already have a starting value;
use [flow](#flow-ƒ) to define a reusable composed function with no value yet.

```typescript

function pipe<A, B>(value: A, ab: (a: A) => B): B
function pipe<A, B, C>(value: A, ab: (a: A) => B, bc: (b: B) => C): C
function pipe<A, B, C, D>(value: A, ab: (a: A) => B, bc: (b: B) => C, cd: (c: C) => D): D
function pipe<A, B, C, D, E>(value: A, ab: (a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E): E
function pipe<A, B, C, D, E, F>(value: A, ab: (a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E, ef: (e: E) => F): F
function pipe<A, B, C, D, E, F, G>(value: A, ab: (a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E, ef: (e: E) => F, fg: (f: F) => G): G
function pipe<A, B, C, D, E, F, G, H>(value: A, ab: (a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E, ef: (e: E) => F, fg: (f: F) => G, gh: (g: G) => H): H
function pipe<A, B, C, D, E, F, G, H, I>(value: A, ab: (a: A) => B, bc: (b: B) => C, cd: (c: C) => D, de: (d: D) => E, ef: (e: E) => F, fg: (f: F) => G, gh: (g: G) => H, hi: (h: H) => I): I
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `value` | `A` |  |

#### Example

```ts

  parseConfig(raw),
```

### recover <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

```typescript
function recover<T, E, F>(result: Result<T, E>, fn: (error: E) => Result<T, F>): Result<T, F>
function recover<T, E, F>(fn: (error: E) => Result<T, F>): (result: Result<T, E>) => Result<T, F>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |
| `fn` | `(error: E) => Result<T, F>` |  |

#### Example

```ts
recover(networkResult, (e) =>
  e.kind === "rate_limit" ? ok(cachedBody) : err(e),
);
```

### trySync <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Wraps a throwing sync function, returning a function that produces a
[Result](#result-type) instead of throwing. Thrown errors pass through `onThrow` to
become a typed `Err`; a normal return becomes `Ok`. The neverthrow analogue
is `Result.fromThrowable`.

```typescript
function trySync<A extends readonly unknown[], T, E>(fn: (...args: A) => T, onThrow: (error: unknown) => E): (...args: A) => Result<T, E>
function trySync<F extends (...args: never) => unknown, E>(fn: F, onThrow: (error: unknown) => E): (...args: Parameters<F>) => Result<ReturnType<F>, E>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `fn` | `(...args: A) => T` | - the throwing function to wrap |
| `onThrow` | `(error: unknown) => E` | - maps a thrown value to the `Err` channel |

#### Returns

a function with `fn`'s parameters that returns `Result<ReturnType, E>`

#### Example

```ts
type ParseError = { kind: "parse"; message: string };
const parse = trySync(
  JSON.parse,
  (e): ParseError => ({ kind: "parse", message: String(e) }),
);
parse("{}");      // Ok({})
parse("nope");    // Err({ kind: "parse", … })
```

### unwrapErr <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Test/assert helper — returns the `Err` value, or **throws a `TypeError`**
when called on an `Ok`. Mirror of [unwrapOk](#unwrapok-ƒ) on the error track and
part of the same assertion tier (RFC 0001 §4): intended for `*.spec.ts` /
`*.test.ts`. Prefer [match](./pattern#match-ƒ) / [unwrapOr](./maybe#unwrapor-ƒ) in business logic; the
lint plugins flag `unwrapErr` outside test files.

```typescript
function unwrapErr<T, E>(result: Result<T, E>): E
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |

#### Returns

the unwrapped `Err` value

#### Example

```ts
// in a *.spec.ts
expect(unwrapErr(err("x"))).toBe("x");
expect(() => unwrapErr(ok(5))).toThrow(TypeError);
```

the unwrapped `Ok` value

### ResultAsync <span className="inline-flex items-center rounded-full border border-blue-200 dark:border-blue-800 bg-blue-50 dark:bg-blue-950/30 text-blue-600 dark:text-blue-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">class</span>

Async railway carrier. Wraps a deferred `Promise<Result<T, E>>` and exposes
the same dual-track transforms as the sync [Result](#result-type) — `map`, `flatMap`,
`recover`, `tap`, `match`. The public API never surfaces `Promise<Result<…>>`
directly: `await` the instance (it is thenable) or call [resolve](#resolve) /

```typescript
class ResultAsync
```

#### Example

```ts
const user = ResultAsync.fromPromise(api.getUser(id), toAppError)
  .map((u) => u.profile)
  .recover(() => ResultAsync.ok(guestProfile));

const r = await user;          // Result<Profile, AppError>
```

(first failure wins). For heterogeneous tuples that preserve per-index

```typescript
function combine<T, E>(results: readonly Result<T, E>[]): Result<T[], E>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `results` | `readonly Result<T, E>[]` |  |

#### Example

```ts
combine([ok(1), ok(2)]);        // Ok([1, 2])
combine([ok(1), err("e")]);     // Err("e")
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### combineTuple <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Heterogeneous tuple combine — like [combine](#combine-ƒ) but preserves each

|---|---|---|
| `results` | `R` |  |

```ts
  const [n, s] = r.value;   // typed per position

```

### asyncAfter <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Bind a sync [Result](#result-type) into an async step without widening defects.
Dual-form: data-first for one-shots, data-last (curried) for `pipe`/`flow`.

```typescript
function asyncAfter<T, U, E, F>(result: Result<T, E>, fn: (value: T) => ResultAsync<U, F>): ResultAsync<U, E | F>
function asyncAfter<T, U, F>(fn: (value: T) => ResultAsync<U, F>): <E>(result: Result<T, E>) => ResultAsync<U, F | E>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |
| `fn` | `(value: T) => ResultAsync<U, F>` |  |

#### Example

```ts
asyncAfter(result, (u) => tryAsync(db.insert(u), toErr)); // data-first
asyncAfter((u) => tryAsync(db.insert(u), toErr));         // data-last
```

### fromAsync <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Lifts a `(...args) => Promise<Result<T, E>>` function (typical of interop or
boundary code) into one returning `(...args) => ResultAsync<T, E>`. Expected
`Err`s pass through; an unexpected promise rejection (a defect) is routed via
`onDefect`, defaulting to [UnexpectedError](#unexpectederror-class) and widening the error union.

```typescript
function fromAsync<A extends readonly unknown[], R extends AnyResult>(fn: (...args: A) => Promise<R>, onDefect?: ((error: unknown) => UnexpectedError | InferErr<R>) | undefined): (...args: A) => ResultAsync<...>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `fn` | `(...args: A) => Promise<R>` | - a function returning a promise that already yields a `Result` |
| `onDefect` | `((error: unknown) => UnexpectedError \| InferErr<R>) \| undefined` | - maps an unexpected rejection to the `Err` channel |

#### Example

```ts
const loadUser = fromAsync(
  (id: string): Promise<Result<User, NotFound>> => api.getUser(id),
);
const ra = loadUser("u1");   // ResultAsync<User, NotFound | UnexpectedError>
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### fromResult <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Lifts an already-settled sync [Result](#result-type) into a [ResultAsync](#resultasync-class), so
it can be chained alongside async steps in a railway.

```typescript
function fromResult<T, E>(result: Result<T, E>): ResultAsync<T, E>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `result` | `Result<T, E>` |  |

#### Example

```ts
const ra = fromResult(ok(1));            // ResultAsync<number, never>

```

### yieldResult <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Unwraps a [Result](#result-type) inside a [tryGen](#trygen-ƒ) block — the sync analogue of
Rust's `?` operator. On `Ok` it returns the value; on `Err` it short-circuits
the enclosing `tryGen`, which returns that `Err`. Only valid inside `tryGen`;
prefer [flatMap](#flatmap-ƒ) / `pipe` for long pipelines.

```ts
});
```

### Ok <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

### Result <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

The core railway type: a tagged union carrying either a success value
(`Ok<T>`) or a failure value (`Err<E>`). It is a plain discriminated union
on `_tag` — no classes — so it is immutable and tree-shake friendly. Narrow
with [isOk](#isok-ƒ) / [isErr](#iserr-ƒ) to read `.value` / `.error`.

```typescript
export type Result<T, E> =
  | { readonly _tag: "Ok"; readonly value: T }
  | { readonly _tag: "Err"; readonly error: E };
```

#### Example

```ts
function parse(raw: string): Result<number, "nan"> {
  const n = Number(raw);
  return Number.isNaN(n) ? err("nan") : ok(n);
}
```
