# @onrails/pattern API Reference

## Matching

### match <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Start a pattern-matching expression. Two call shapes:

- `match(value)` — data-first; subsequent `.exhaustive()` or
  `.otherwise(fn)` runs immediately against `value`.
- `match<T>()` — curried; subsequent `.exhaustive()` returns a function
  `(value: T) => R` for use in `pipe(...)` or as a reusable matcher.

```typescript
function match<T>(input: T): MatchBuilder<T, never, true, [], false>
function match<T = never>(): MatchBuilder<T, never, false, [], false>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `input` | `T` |  |

#### Example

```ts
// Data-first
const out = match({ kind: "click", x: 1, y: 2 } as Event)
  .with({ kind: "click" }, (e) => e.x + e.y)
  .with({ kind: "key" },   () => -1)
  .exhaustive();

// Curried — build a reusable matcher
const describe = match<Event>()
  .with({ kind: "click" }, (e) => `click @ ${e.x},${e.y}`)
  .with({ kind: "key" },   (e) => `key ${e.code}`)
  .exhaustive();
```

### MatchBuilder <span className="inline-flex items-center rounded-full border border-blue-200 dark:border-blue-800 bg-blue-50 dark:bg-blue-950/30 text-blue-600 dark:text-blue-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">class</span>

Fluent builder for [match](./result#match). Tracks the handled cases at the type
level so `.exhaustive()` only compiles when every union member is covered.

The `Locked` phantom flag (set via `.returnType<R>()`) constrains every
subsequent handler to return `R`, useful when branch return-type
inference would widen to a union narrower than the target slot
(e.g. `ReactNode`, an API DTO).

```typescript
class MatchBuilder
```

#### Example

```ts
const describe = match<Event>()
  .with({ kind: "click" }, (e) => `click @ ${e.x},${e.y}`)
  .with({ kind: "key" },   (e) => `key ${e.code}`)
  .exhaustive();
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

```typescript
constructor<T, R = never, HasInput extends boolean = false, Handled extends readonly unknown[] = [], Locked extends boolean = false>(cases: readonly Case<T, unknown>[], input: T | typeof NO_INPUT, _handled?: Handled): MatchBuilder<T, R, HasInput, Handled, Locked>
```

### MatchBuilder.prototype.with <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Add a case: when `input` satisfies `pattern`, run `handler` on the narrowed
value. The pattern may be a literal, a shallow object pattern, or a guard
([Pattern](#pattern)); a type-predicate guard narrows the handler input. Each
`.with` advances compile-time exhaustiveness tracking so `.exhaustive()`
knows which union members remain.

```typescript
with<const P extends Pattern<T>, R2>(pattern: P, handler: (input: Narrow<T, P>) => HandlerReturn<Locked, R, R2>): MatchBuilder<T, NextResult<Locked, R, R2>, HasInput, readonly [......], Locked>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `pattern` | `P` | - literal, object pattern, or guard to test against |
| `handler` | `(input: Narrow<T, P>) => HandlerReturn<Locked, R, R2>` | - run on the narrowed input when the pattern matches |

#### Returns

a builder with this case appended

#### Example

```ts
type Event =
  | { type: "message"; content: string }
  | { type: "error"; message: string }
  | { type: "done" };

const summary = match({ type: "message", content: "hi" } as Event)
  .with({ type: "message" }, (e) => e.content)   // e: { type: "message"; content: string }
  .with({ type: "error" },   (e) => e.message)
  .with({ type: "done" },    () => "—")
  .exhaustive();                                 // "hi"
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### MatchBuilder.prototype.withOneOf <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

One handler for several patterns (OR). The handler input is the union of
the members narrowed by each pattern, and all of them are marked handled
for exhaustiveness.

```typescript
withOneOf<const Ps extends readonly Pattern<T>[], R2>(patterns: Ps, handler: (input: NarrowUnion<T, Ps>) => HandlerReturn<Locked, R, R2>): MatchBuilder<T, NextResult<...>, HasInput, readonly [......], Locked>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `patterns` | `Ps` | - patterns to test; any match runs `handler` |
| `handler` | `(input: NarrowUnion<T, Ps>) => HandlerReturn<Locked, R, R2>` | - run on the union of narrowed members |

#### Returns

a builder with the shared case appended

#### Example

```ts
type Job =
  | { kind: "queued"; id: string }
  | { kind: "running"; id: string }
  | { kind: "done"; ok: boolean };

const status = match({ kind: "running", id: "j1" } as Job)
  .withOneOf([{ kind: "queued" }, { kind: "running" }], (j) => j.id) // j: queued | running
  .with({ kind: "done" }, (j) => (j.ok ? "yes" : "no"))
  .exhaustive();                                                      // "j1"
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### MatchBuilder.prototype.withEither <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

[withOneOf](#withoneof) for exactly two patterns — sugar that avoids the array.

```typescript
withEither<const P1 extends Pattern<T>, const P2 extends Pattern<T>, R2>(pattern1: P1, pattern2: P2, handler: (input: NarrowUnion<T, readonly [P1, P2]>) => HandlerReturn<Locked, R, R2>): MatchBuilder<...>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `pattern1` | `P1` | - first pattern |
| `pattern2` | `P2` | - second pattern |
| `handler` | `(input: NarrowUnion<T, readonly [P1, P2]>) => HandlerReturn<Locked, R, R2>` | - run on the union of the two narrowed members |

#### Returns

a builder with the shared case appended

#### Example

```ts
type Provider = "ollama" | "openrouter";

const pick = match<Provider>()
  .withEither("ollama", "openrouter", (p) => p) // p: "ollama" | "openrouter"
  .exhaustive();
pick("ollama"); // "ollama"
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### MatchBuilder.prototype.returnType <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Lock the result type. All subsequent `.with()` handlers must return `R2`.
Useful when branch return-type inference widens to a union narrower than
the slot the match feeds into (e.g. `ReactNode`).

Also the fix for literal-widening: when a handler returns an object whose
field is a union literal, inference widens it (e.g. `{ mode: "native" }`
becomes `{ mode: string }`, breaking a `Mode` union). Lock the target type
instead of sprinkling `as const`:

```typescript
returnType<R2>(): MatchBuilder<T, R2, HasInput, Handled, true>
```

#### Example

```ts
type State = { mode: "compat" | "native" };
const next = match(flag)
  .returnType<State>()                       // handlers now checked against State
  .with("--native", () => ({ mode: "native" })) // no `as const` needed
  .otherwise(() => state);
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### MatchBuilder.prototype.run <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Run the accumulated cases now, without a compile-time exhaustiveness check.
Uses the value passed to [match](./result#match) (data-first), or the value passed to
`run` (curried). Throws `"Non-exhaustive match"` if no case matches at
runtime — prefer [exhaustive](#exhaustive) when the input is a closed union, or
[otherwise](#otherwise) for an explicit fallback.

```typescript
run(...input: readonly [input: T] | readonly []): R
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `input` | `readonly [input: T] \| readonly []` | - the value to match when the builder was created curried |

#### Returns

the matched handler's result

#### Example

```ts
type Event = { type: "error"; message: string } | { type: "done" };

const msg = match({ type: "error", message: "x" } as Event)
  .with({ type: "error" }, (e) => e.message)
  .run({ type: "error", message: "net" }); // "net"
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### MatchBuilder.prototype.exhaustive <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Settle the match with a compile-time exhaustiveness guarantee. Only
type-checks when every member of `T` has been handled; otherwise the return
type becomes [NonExhaustiveError](#nonexhaustiveerror), surfacing the missing cases as a
type error. Returns the result directly when data-first ([match](./result#match)
called with a value), or a reusable `(input: T) => R` matcher when curried.

```typescript
exhaustive(): ExhaustiveResult<T, Handled, R, HasInput>
```

#### Returns

the result (data-first) or a matcher function (curried)

#### Example

```ts
type Event =
  | { type: "message"; content: string }
  | { type: "error"; message: string }
  | { type: "done" };

// Curried — reusable matcher; omitting a `.with` would be a type error.
const describe = match<Event>()
  .with({ type: "message" }, (e) => e.content)
  .with({ type: "error" },   (e) => `! ${e.message}`)
  .with({ type: "done" },    () => "done")
  .exhaustive();
describe({ type: "done" }); // "done"
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### MatchBuilder.prototype.otherwise <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Settle the match with a catch-all fallback, dropping the exhaustiveness
requirement — `handler` runs (on the full input type) when no case matched.
Returns the result directly when data-first, or a reusable matcher when
curried.

```typescript
otherwise(handler: (input: T) => R): HasInput extends true ? R : (input: T) => R
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `handler` | `(input: T) => R` | - fallback run on the input when no prior case matched |

#### Returns

the result (data-first) or a matcher function (curried)

#### Example

```ts
type Event =
  | { type: "message"; content: string }
  | { type: "error"; message: string }
  | { type: "done" };

const len = match({ type: "done" } as Event)
  .with({ type: "message" }, (e) => e.content.length)
  .otherwise(() => -1); // -1
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### matchTag <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Exhaustive match on `_tag` — for `@onrails/result` / `@onrails/maybe` style
unions. The `handlers` object must supply one branch per `_tag`, and each
handler receives the member narrowed to its tag.

```typescript
function matchTag<T extends Tagged, R>(value: T, handlers: TagHandlers<T, R>): R
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `value` | `T` | - a tagged union value to dispatch on |
| `handlers` | `TagHandlers<T, R>` | - one handler per `_tag`, keyed by tag |

#### Returns

the result of the matched handler

#### Example

```ts
import { err, ok, type Result } from "@onrails/result";

const label = (r: Result<number, string>) =>
  matchTag(r, {
    Ok:  (v) => `ok:${v.value}`,  // v: Ok<number>
    Err: (e) => `err:${e.error}`, // e: Err<string>
  });
label(ok(1));    // "ok:1"
label(err("x")); // "err:x"
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### when <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Guard pattern for [match](./result#match).with.

Accepts plain boolean guards (`(x) => x.length > 0`) or TS type-predicate
guards (`(x): x is Foo => ...`). For type predicates, the matched handler's
input is narrowed to the predicate's target type via [Narrow](#narrow).

Note: a plain boolean guard does not advance exhaustiveness tracking — pair
it with `.otherwise()`, or use a type predicate to narrow the union.

```typescript
function when<T, U extends T>(guard: (input: T) => input is U): (input: T) => input is U
function when<T>(guard: (input: T) => boolean): Pattern<T>
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `guard` | `(input: T) => input is U` | - boolean or type-predicate guard function |

#### Returns

the guard, usable as a [Pattern](#pattern) in `.with`

#### Example

```ts
const sign = match<number>()
  .with(when((n) => n < 0), () => "neg")
  .with(when((n) => n >= 0), () => "non-neg")
  .exhaustive();
sign(-1); // "neg"
sign(0);  // "non-neg"
```


## Diagnostics

### assertNever <span className="inline-flex items-center rounded-full border border-emerald-200 dark:border-emerald-800 bg-emerald-50 dark:bg-emerald-950/30 text-emerald-600 dark:text-emerald-400 text-xs font-semibold px-2.5 py-0.5 font-medium ml-2 align-middle">ƒ</span>

Use in `default` branches (or after manual narrowing) to assert a union has
been fully handled. If every case is handled, `value` is `never` and the call
type-checks; an unhandled case makes `value` non-`never`, surfacing a compile
error. Always throws at runtime.

```typescript
function assertNever(value: never, message?: string): never
```

#### Parameters

| Parameter | Type | Description |
|---|---|---|
| `value` | `never` | - the value that should be unreachable (typed `never`) |
| `message` | `string` | - error message prefix; defaults to `"Unreachable"` |

#### Example

```ts
type Shape = { kind: "circle"; r: number } | { kind: "square"; s: number };

function area(shape: Shape): number {
  switch (shape.kind) {
    case "circle": return Math.PI * shape.r ** 2;
    case "square": return shape.s ** 2;
    default: return assertNever(shape); // type error if a case is added
  }
}
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### NonExhaustiveError <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

Returned by `.exhaustive()` when union cases are still missing (compile-time error).

```typescript
export type NonExhaustiveError<Remaining> = {
  readonly __nonExhaustive: "Add .with() branches for remaining cases";
  readonly remaining: Remaining;
};
```


## Types

### ExhaustiveResult <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

`.exhaustive()` return type — `NonExhaustiveError` when cases are missing.

```typescript
export type ExhaustiveResult<T, Handled extends readonly unknown[], R, HasInput extends boolean> =
  IsExhaustive<T, Handled> extends true
    ? ExhaustiveOutput<R, HasInput, T>
    : NonExhaustiveError<RemainingCases<T, Handled>>;
```

### ExhaustMatched <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

Union members of `T` ruled out by pattern `P`.
Boolean guards (non-predicates) do not advance exhaustiveness — use `when(isX)` or `.otherwise()`.

```typescript
export type ExhaustMatched<T, P> = P extends (input: T) => boolean
  ? [GuardTarget<P>] extends [never]
    ? never
    : Extract<T, GuardTarget<P>>
  : Extract<T, P>;
```

```typescript
export type HandledUnion<Handled extends readonly unknown[]> = Handled[number];
```

### IsExhaustive <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

```typescript
export type IsExhaustive<T, Handled extends readonly unknown[]> = [
```

### LockedMatchBuilder <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

Result-locked variant of [MatchBuilder](#matchbuilder). Constructed via

```typescript
export type LockedMatchBuilder<
```

### Narrow <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

Narrows `input` when `pattern` is a shallow object, literal discriminant, or type predicate.

```typescript
export type Narrow<T, P> = P extends (input: T) => boolean
  ? [GuardTarget<P>] extends [never]
    ? T
    : NarrowObject<T, GuardTarget<P>>
  : P extends Record<string, unknown>
    ? NarrowObject<T, P>
    : T extends P
      ? Extract<T, P>
      : T;
```

### NarrowUnion <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

Union of members narrowed by each pattern in `Ps` (multi-branch handlers).

```typescript
export type NarrowUnion<T, Ps extends readonly unknown[]> = Narrow<T, Ps[number]>;
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### Pattern <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

A pattern that matches a value of type `T`. One of:

- **Literal** — a primitive (`"a"`, `42`, `true`) matched by `===`.
- **Object pattern** — a `Partial<T>` of shallow key/value pairs that
  must all match by strict equality. Only available when `T` is an
  object type.
- **Guard** — a function `(input: T) => boolean`. Type predicates
  (`(x): x is U`) narrow the handler input via [Narrow](#narrow).

```typescript
export type Pattern<T> = T | ObjectPattern<T> | PatternGuard<T>;
```

#### Example

```ts
type Event = { kind: "click"; x: number } | { kind: "key"; code: string };

const p1: Pattern<Event> = { kind: "click" };         // object pattern
const p2: Pattern<Event> = (e) => e.kind === "key";   // guard
```

<hr className="my-8 border-neutral-200 dark:border-neutral-800" />

### RemainingCases <span className="inline-flex items-center rounded-full border border-teal-200 dark:border-teal-800 bg-teal-50 dark:bg-teal-950/30 text-teal-600 dark:text-teal-400 uppercase tracking-wider px-2 py-0.5 text-[10px] font-medium ml-2 align-middle">type</span>

Input cases not yet covered by `.with` / `.withOneOf` / `.withEither`.

```typescript
export type RemainingCases<T, Handled extends readonly unknown[]> = Exclude<
  T,
  HandledUnion<Handled>
>;
```
