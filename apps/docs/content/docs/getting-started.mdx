### Install

Add the three primitives — `Result`, `Maybe`, and exhaustive `match`. They're
independent; install only what you need.

<Tabs items={["bun", "npm", "pnpm", "yarn"]}>

<Tab value="bun">

```bash
bun add @onrails/result @onrails/maybe @onrails/pattern
```

</Tab>

<Tab value="npm">

```bash
npm install @onrails/result @onrails/maybe @onrails/pattern
```

</Tab>

<Tab value="pnpm">

```bash
pnpm add @onrails/result @onrails/maybe @onrails/pattern
```

</Tab>

<Tab value="yarn">

```bash
yarn add @onrails/result @onrails/maybe @onrails/pattern
```

<Step>

### Combine a flow with pattern matching

A fallible calculation returns a `Result`; matching collapses both branches into
one value — no thrown exceptions on the expected path.

```typescript
import { Result } from "@onrails/result";
import { match } from "@onrails/pattern";

// Fallible calculation
function parseId(input: string): Result<number, string> {
  const parsed = parseInt(input, 10);
  return isNaN(parsed) ? Result.err("Not a number") : Result.ok(parsed);
}

const outcome = parseId("42")
  .map(num => num * 2)
  .match(
    (val) => `Success: ${val}`,
    (err) => `Error: ${err}`
  );

console.log(outcome); // "Success: 84"
```

<Callout type="warn">
  Keep `Result` at boundaries: don't return `Promise<Result<…>>` from public
  signatures — return `ResultAsync<T, E>` instead. `@onrails/eslint-plugin` and
  `@onrails/biome-plugin` enforce this for consumers.
</Callout>
