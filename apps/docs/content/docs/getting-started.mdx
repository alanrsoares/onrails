### Return a Result instead of throwing

A fallible function returns a `Result<T, E>` — `ok` on the happy path, a typed
`err` on the expected failure. The core is **data-first**: import the package as
a namespace (`R`) and pass the result as the first argument. `map` transforms the
`Ok` channel; `match` collapses both branches into one value — no thrown
exceptions on the expected path.

```ts twoslash
import * as R from "@onrails/result";

// Fallible calculation — Ok or a typed Err, never a throw.
function parseId(input: string): R.Result<number, string> {
  const n = Number.parseInt(input, 10);
  return Number.isNaN(n) ? R.err("Not a number") : R.ok(n);
}

const outcome = R.match(
  R.map(parseId("42"), (n) => n * 2),
  (val) => `Success: ${val}`,
  (err) => `Error: ${err}`,
);

console.log(outcome);
//          ^?
```

<Callout title="Data-first, not fluent">
  The sync core is data-first: `R.map(result, fn)`, not `result.map(fn)`. Reach
  for [`pipe` / `flow`](/docs/recipes) to chain steps point-free. `ResultAsync`
  *is* fluent (`.flatMap`, `.match`) — see below.
</Callout>
