# @onrails/docgen

Internal docs generators for the onrails documentation site. **Private** — not
published; consumed by `apps/docs` and the root `docs:generate` script.

Two generators, both project-agnostic engines configured by thin runners:

- **snippets** — extract `#region snippet` blocks from example modules into a
  generated module with display + [twoslash](https://twoslash.netlify.app/)
  forms.
- **api** — turn TypeScript source + JSDoc/TSDoc into MDX API-reference pages.

The engines hold no onrails specifics; paths, import rewriting, symbol
categorization, and cross-package linking are passed in as options. The onrails
configuration lives in the runners (`apps/docs/scripts/gen-snippets.ts` and
`scripts/generate-api-docs.ts`).

## Entry points

| Import | Exports |
| --- | --- |
| `@onrails/docgen` | everything below |
| `@onrails/docgen/snippets` | `extractSnippets`, `buildSnippetsModule` |
| `@onrails/docgen/api` | `generateApiDocs` |

## Snippets

`extractSnippets` scans a directory for `*.ts` modules, pulls the
`#region snippet` … `#endregion` block from each, and writes a generated module
keyed by file name. Every entry has two forms:

- `code` — display form: region only, `export` stripped, fixture imports
  dropped, imports optionally rewritten to package specifiers.
- `twoslash` — a self-contained module: shown imports on top, fixtures hidden
  inside `// ---cut-start---` / `// ---cut-end---`, body below — so twoslash
  type-checks the whole thing but renders only the snippet, with hover types.

```ts
import { extractSnippets } from "@onrails/docgen/snippets";

await extractSnippets({
  srcDir: "packages/examples/src",
  outFile: "apps/docs/lib/snippets.generated.ts",
  // optional: rewrite relative source imports to published specifiers
  rewriteImport: (line) =>
    line.replace(/from "(?:\.\.\/)+([\w-]+)\/src\/index\.js"/, 'from "@scope/$1"'),
});
```

`buildSnippetsModule(modules, fixturesSource, opts)` is the pure core (no disk
access) behind `extractSnippets` — unit-testable, returns `{ module, ids, skipped }`.

### Options

| Option | Default | Purpose |
| --- | --- | --- |
| `srcDir` | — | directory scanned for `*.ts` snippet modules |
| `outFile` | — | path of the generated module to write |
| `fixtureName` | `"fixtures"` | basename of the shared fixtures module in `srcDir` |
| `rewriteImport` | identity | rewrites a single import line |
| `sourceLabel` | `srcDir` | label in the generated-file header comment |
| `generatedBy` | `"extract-snippets"` | attribution in the `// Generated by …` header |

## API reference

`generateApiDocs(packages, opts)` parses each package entrypoint, extracts its
exported symbols (signatures, JSDoc, `@param`/`@returns`/`@example`/`@deprecated`,
class members), and writes one MDX file per package. All packages are parsed
first to build a cross-package export map used for `{@link}` resolution.

```ts
import { generateApiDocs } from "@onrails/docgen";

generateApiDocs(
  [
    { entry: "packages/result/src/index.ts", name: "@onrails/result", out: "…/result.mdx" },
    { entry: "packages/maybe/src/index.ts", name: "@onrails/maybe", out: "…/maybe.mdx" },
  ],
  {
    // all optional — sensible defaults ship in the package
    categorize: (name, pkg, tags) => /* … */ undefined,
    categoryOrder: { "@onrails/result": ["Core", "Async", "Types"] },
    resolveLink: (symbol, currentPackage, exports) => /* … */ "#anchor",
  },
);
```

### Options

| Option | Default | Purpose |
| --- | --- | --- |
| `categorize` | reads `@category`, else `"Core"` | symbol → category |
| `categoryOrder` | alphabetical | preferred category order per package |
| `resolveLink` | local anchor → sibling page → local | resolve `{@link X}` targets |

The default renderer emits Fumadocs-flavored MDX with Tailwind-styled badges.

## Develop

```bash
bun run --filter @onrails/docgen check   # typecheck + test
```

The package runs from source via Bun (no build step). Output is verified by the
consuming gates: `apps/docs` regenerates both artifacts on `predev`/`prebuild`
and type-checks them, and the example modules are tested by the monorepo gate —
so every rendered snippet compiles against the real API.
